Spacing multiplier

--spacing: 0.3125rem (5 px) replaces the default 0.25rem (4 px). Every w-*, p-*, m-*, and gap-* utility is derived from this single multiplier via calc(var(--spacing) * n).

w-1 = 5 px default: 4 px
w-2 = 10 px default: 8 px
w-4 = 20 px default: 16 px
w-8 = 40 px default: 32 px
w-16 = 80 px default: 64 px
w-32 = 160 px default: 128 px
Recommended when: your design system or Figma file is built on a non-4 px grid (e.g. 5 px or 8 px). Aligning the multiplier once removes the mental conversion on every spacing value. Avoid in existing projects — it is a breaking change for every component already using spacing utilities.

Border radius scale

Overriding the --radius-* tokens shifts every rounded-* utility globally — change the scale once and every component using it updates automatically.

rounded-sm 4 px default: 2 px
rounded-md 8 px default: 6 px
rounded-lg 12 px default: 8 px
rounded-xl 20 px default: 12 px
rounded-2xl 32 px default: 16 px
rounded-3xl 24 px unchanged
rounded-full 9999 px hardcoded, not a token

Color step override

--color-blue-500 is replaced with a sky-toned value. All neighbouring blue steps remain at their Tailwind defaults — note the visual discontinuity at the overridden step.

blue-200 default
blue-300 default
blue-400 default
blue-500 overridden
blue-600 default
blue-700 default
blue-900 default
Raw step override vs. semantic alias. Overriding --color-blue-500 replaces it everywhere — including third-party components you don't control. The visual gap between blue-400 and blue-500 above illustrates the inconsistency this can introduce. For brand customisation, prefer semantic aliases (lesson 03): declare --color-brand: var(--color-blue-500) and reference only bg-brand in your components.
